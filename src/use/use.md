# MaiCore Start 启动指南(萌新版)

## 👶 新手快速上手

> **第一次使用？不用担心！** 按照下面的简单步骤，30分钟就能启动您的第一个 Bot 实例。

## 第零步：<mark>**最最最最重要的一点！！！获取你的API秘钥！！！**</mark>

你可认为它是驱动Bot运行的燃料，没有它，你的Bot就无法运行！
我们默认使用硅基流动的API秘钥，使用硅基流动的服务，
> API ，你可以理解为这是一个应用去请求另一个应用的某个功能的通道，在这里我们就是通过 API 去请求硅基流动的 AI 模型的。 
1. [点击这里注册硅基流动的账号。](https://cloud.siliconflow.cn/i/JSydmfX7)
2. 点击侧边栏的**实名认证**选项卡完成实名认证。
3. 点击侧边栏的**API秘钥**选项卡，点击 **🔑新建API秘钥** 申请一个你的API秘钥，完成后你会获得一串`sk-`开头的字母，那个就是API秘钥，稍后我们在配置Bot时会用到。


## 第一步：启动程序 🚀

1. **双击运行** `MaiCoreStart`
2. 等待程序加载完成，会显示彩色的终端界面，<mark>最新版的程序加入了新手引导功能，跟着它做就好</mark>

这个就是主菜单：

```

```

## 第二步：部署新实例（推荐新手）🆕

如果您是第一次使用，选择 `[F] 📦 实例部署辅助系统` → `[A] 实例部署`：

1. **选择 Bot 类型**
   ```
   🤖 Bot类型选择
   请选择要部署的Bot类型：
    [1] MaiBot (默认)  ← 推荐新手选择这个
    [2] MoFox_bot
   ```
   **新手建议**: 直接按回车选择 `1` (MaiBot)

2. **选择版本**
   - 程序会自动获取最新版本，如果失败了，就等一等再手动重试
   - 推荐选择最新的 **发行版**（稳定版本），或者 **`main/master`** 分支
   - 如果不确定，直接选择<mark>第一个</mark>选项

3. **选择组件**（根据提示回答）
   - **适配器**: 建议选择 `Y`（需要）
   - **NapCat**: 建议选择 `Y`（需要，用于连接QQ）
   - **MongoDB**: 根据版本选择，通常选择 `N`
   - **WebUI**: 建议选择 `Y`（网页管理界面）

4. **设置安装路径**
   - 建议输入您想要的路径
   ::: warning
   路径中不能有中文哦
   :::

5. **设置实例名称**
   - 给您的 Bot 起个名字，比如：`newBot`

6. **设置序列号**
   - 输入一个简单的数字，或者直接用你的bot名字比如：`newbot`
   - 用于区分不同的 Bot 实例

7. **设置QQ号**（可选）
   - 输入您要用于机器人的QQ号
   - 如果暂时没有，可以直接回车跳过

8. **配置您的bot的人格**
   - 部署完成后，我们会询问您是否在文本编辑器中打开主要配置文件，默认使用 VSCode 如果你没安装，则会在记事本中打开，建议在部署前先[在组件下载中安装 VSCode ](../function/miscellaneous/components)，<mark>我还是比较推荐你打开的，免得你找不着</mark>，如果您部署的是麦麦bot并且部署了控制面板（WebUI），那么您可以在控制面板中对麦麦bot的人格进行编辑。
   - [MaiBot 配置与安装简介](https://docs.mai-mai.org/manual/configuration/)
   - [MoFox-Bot 配置文件 (bot_config.toml) 究极详细指南](https://docs.mofox-sama.com/docs/guides/bot_config_guide.html)
   - 这时我们就要用到先前获取的 API 秘钥了，在`model_config.toml`中，你可以找到这一栏：
   ```toml
   [[api_providers]] # SiliconFlow的API服务商配置
   name = "SiliconFlow"
   base_url = "https://api.siliconflow.cn/v1"
   api_key = "your-siliconflow-api-key"  #你需要做的，就是用你新获得的 API 秘钥替换掉`your-siliconflow-api-key`，请务必保留引号""
   client_type = "openai"
   max_retry = 3
   timeout = 120
   retry_interval = 5
   ```

## 第三步：启动 Bot 🤖

部署完成后，返回主菜单，选择 `[A] 🚀 运行实例`：

1. **选择要启动的实例**
   - 输入实例的序列号（比如 `newbot`）
   - 或者从列表中选择你要启动的实例

2. **选择启动方式**
   ```
   [预设启动项]
    [1] 主程序+适配器
    [2] 主程序+适配器+NapCatQQ
    [3] 主程序+适配器+检查MongoDB
    [4] 主程序+适配器+NapCatQQ+检查MongoDB
    [5] 主程序+适配器+控制面板
    [6] 主程序+适配器+NapCat+控制面板
    [H] 高级启动项
    [Q] 返回
   请选择启动方式:  ():
   ```
   **新手建议**: 选择 `[6] 主程序+适配器+NapCat+控制面板`

3. **等待启动完成**
   - 程序会自动启动所有组件
   - 看到这个 ↓ 就是启动成功啦
   ```
   ✅ 🎉 启动流程完成！
   ✅ 启动操作完成！
   按回车键继续...
   ```


## 第四步：访问管理界面 🌐

启动成功后，控制面板会自动在默认浏览器打开

- **登录麦麦控制面板**: 访问 `http://localhost:7999`（如果安装了WebUI）
   1. 仔细查看麦麦主程序的终端窗口
   2. 找到像这样的输出：
      ```
      12-17 22:51:55 [webui] 新的 WebUI Token 已生成: 49c43a37...
      12-17 22:51:55 [webui_server] 🔑 WebUI Access Token: 49c43a37174b7ff07ce0d8231429837455747ba2702fee241c7911cc80b35e51
      12-17 22:51:55 [webui_server] 💡 请使用此 Token 登录 WebUI
      ```
      复制 WebUI Access Token ，在这里就是`49c43a37174b7ff07ce0d8231429837455747ba2702fee241c7911cc80b35e51`
   3. 回到控制面板的登录界面并在 Access Token 处填入这一大串，回车，登录

- **登录NapCatWebUI（Shell版）**
  1. NapCat 启动成功后，仔细查看终端窗口，找到类似这样的输出：
    ```
   12-18 17:27:21 [info] [NapCat] [WebUi] WebUi User Panel Url: http://127.0.0.1:6099/webui?token=ccfee65cfff7
   12-18 17:27:21 [info] [NapCat] [WebUi] WebUi User Panel Url: http://0.0.0.0:6099/webui?token=ccfee65cfff7
   ```
   这就是WebUI的登录地址，之所以推荐Shell版，是应为现在NapCat在部署完成后，会自动生成复杂的登录令牌（也就是地址后面的token值），而Shell会直接给出来，方便你登录。

   2. 在浏览器输入 WebUI 的登录地址，回车，选择你要快捷登录的QQ账号或扫码登录。

   3. 登录成功后，选择侧边栏的网络配置 → 新建 → Websocket客户端 

   4. URL填写：`ws://localhost:8095`

   5. 清空Token栏（这是防止你忘记在适配器中填写您的Token）

   6. 将“启用"勾选上

   7. 保存

## 常见新手问题 ❓

### Q: 提示"配置验证失败"怎么办？
**A**: 检查以下几点：
- Bot 路径是否正确
- 所有必需的文件是否存在
- 端口是否被其他程序占用

### Q: 启动时提示端口被占用？
**A**:
- 关闭其他可能占用端口的程序
- 或者使用"实例多开功能"自动分配其他端口

### Q: 组件启动失败？
**A**:
- 检查 Python 环境是否正确安装
- 确认所有依赖包已安装
- 查看具体的错误信息

### Q: 如何停止 Bot？
**A**:
- 在主菜单中选择 ` [G] 📊 查看运行状态`
- 选择您的实例后，输入指令`stop <你要停止的进程的PID>`
- 或者简单粗暴，直接关闭实例的终端窗口
