## 修改日志保留天数
默认情况下，MaiCore-Start会自动删除30天前的日志文件。你可以在程序设置中修改此设置。
> 30天，应该够了，别告我你延后了30天才去翻错误日志。默认30天
## 修改退出时操作
默认情况下，MaiCore-Start会询问是否退出程序。你可以在程序设置中修改此设置，你可以选择在退出时关闭所有托管进程或保留所有托管进程，关闭窗口导致的程序强制退出将不会触发任何对托管进程的操作。
> 默认询问
## 修改颜色 (输入选项数字)
程序终端的打印颜色，不建议乱动，当然，你如果喜欢，你可以修改颜色。
> 默认....（不就在这篇文档的开头吗？）
## 切换最小化到托盘
默认情况下，MaiCore-Start不会最小化到系统托盘。你可以在程序设置中修改此设置，当你开启此项时，你在主菜单选择`[Q] 退出程序`时，程序会最小化到托盘，而不是退出程序，此时不会触发程序退出时才会触发的对托管进程的操作。如果你是关闭窗口导致的程序强制退出，即便你开启此项，程序也不会最小化到托盘。
> 没屁用的绯雾功能，默认关闭
## 切换Windows通知
默认情况下，MaiCore-Start在部署实例时会向Windows通知中心推送通知。你可以在程序设置中修改此设置，你可以选择关闭此功能。但我个人觉得挺有用的，所以默认开启。

这个功能只有在你部署实例时才会触发，免得某些人，嫌部署时间太长，跑去打游戏，俩小时后回来发现部署进程崩了，结果白浪费时间。
> 默认开启，都不许给我关😡

## 设置进程监控刷新间隔
默认进程监控刷新设置：
  - 数据刷新间隔: 2.00 秒
  - UI刷新间隔: 0.30 秒
  - 输入轮询间隔: 0.05 秒

你可以在程序设置中修改此设置，你可以选择修改数据刷新间隔、UI刷新间隔和输入轮询间隔，如果你嫌进程监控的界面太卡，那你可以调小一些。
> 默认：
> - 数据刷新间隔: 2.00 秒
> - UI刷新间隔: 0.30 秒
> - 输入轮询间隔: 0.05 秒

## 配置网络代理

程序的代理设置，加速你的部署进程。<mark>但我推荐你用外置代理软件</mark>，不然你到时崩都不知道怎么崩的。

```default

当前代理配置
  状态: ✗ 未启用
  类型: HTTP
  主机: (未设置)
  端口: (未设置)
  用户名: (未设置)
  密码: (未设置)
  排除主机: localhost,127.0.0.1

💡 常见代理软件配置参考
  • Clash:       HTTP    127.0.0.1:7890
  • V2rayN:      SOCKS5  127.0.0.1:10808
  • Shadowsocks: SOCKS5  127.0.0.1:1080

====>>代理操作<<====
 [E] ✨ 启用代理
 [1] 📝 设置代理类型 (HTTP/HTTPS/SOCKS5/SOCKS4)
 [2] 📝 设置代理主机
 [3] 📝 设置代理端口
 [4] 📝 设置用户名 (可选)
 [5] 📝 设置密码 (可选)
 [6] 📝 设置排除主机
 [T] 📊 测试代理连接

====>>快速配置<<====
 [C] 🚀 快速配置 Clash 代理 (127.0.0.1:7890)
 [V] 🚀 快速配置 V2rayN 代理 (127.0.0.1:10808)
 [S] 🚀 快速配置 Shadowsocks 代理 (127.0.0.1:1080)
```
